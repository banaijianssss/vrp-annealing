# VRP 模拟退火算法说明

## CVRP（带容量约束的车辆路径问题）

**问题定义：** 在容量限制与车辆数上限下，为所有客户规划路线，使总行驶距离最小。不考虑时间窗。

**目标函数：** 最小化所有路线的欧氏距离之和。

**约束：**
- 每个客户恰好访问一次
- 每条路线载重不超过车辆容量
- 使用车辆数不超过上限

**算法流程：**

1. **初始解构造**
   - Clarke-Wright 节约算法：按节约值合并路线，将单客户路线逐步合并为更长的可行路线
   - 最便宜插入：逐个客户插入当前代价增量最小的位置
   - 多种构造结果经强化局部搜索后，取总距离最小的可行解

2. **模拟退火主循环**
   - 初始温度 \(T_0 \approx \text{t0\_factor} \times \text{当前解距离}\)
   - 每个温度下进行若干次迭代，邻域操作包括：
     - 单点 relocate（跨路线移动客户）
     - Or-opt 链 relocate（移动连续 1~3 个客户）
     - Cross-exchange（两路线尾部片段交换，类似 2-opt*）
     - 跨路线 swap（交换两个客户）
     - 路线内随机 2-opt
   - 每次邻域后执行轻量局部搜索（路线内 2-opt + Or-opt）
   - 按 Metropolis 准则接受劣解：以概率 \(e^{-\Delta/T}\) 接受距离更差的解
   - 温度按系数 \(\alpha\) 衰减；长期无改进时额外加速降温

3. **终解强化**
   - 多轮最优单点 relocate + 路线内 2-opt / Or-opt
   - 支持多重启（restarts），取最优结果后再做一次全局强化

**档位参数（随客户数 \(n\) 缩放）：**

| 档位 | max_iter | iter_per_temp | alpha | 重启 |
|------|----------|---------------|-------|------|
| fast | 5000 + 200·n | 50 + 3·n | 0.99 | 1 |
| medium | 8000 + 400·n | 80 + 4·n | 0.995 | 1 |
| high | 20000 + 1000·n | 100 + 10·n | 0.997 | 8 |
| ultra | 40000 + 2000·n | 150 + 15·n | 0.998 | 15 |

## VRPTW（带时间窗的车辆路径问题）

**问题定义：** 在 CVRP 基础上增加时间窗约束。每个客户有就绪时间、截止时间和服务时间；车辆到达早于就绪时间需等待。行驶时间等于欧氏距离。

**目标函数：** 最小化总行驶距离（与 CVRP 相同，可行解优先）。

**约束：**
- CVRP 全部约束
- 每条路线满足 Solomon 时间窗规则：到达不晚于 due，服务不早于 ready，回库不晚于仓库 due

**算法流程：**

1. **初始解构造**
   - **CVRP warm start：** 调用 CVRP 求解器（fast 档）得到容量可行解，作为 VRPTW 候选初解
   - **时间导向插入：** 按 ready / due 排序客户，在每个可行插入位置中选距离增量最小的位置
   - 优先选用时间窗可行的解；若均不可行，选车辆数少、距离短的解交给 SA 改进

2. **模拟退火主循环**
   - 邻域操作：单点 relocate、跨路线 swap、路线内 2-opt（均带时间窗可行性检查）
   - 每次邻域后执行路线内 2-opt 局部搜索（仅接受时间窗可行改进）
   - 退火参数与 CVRP 共用 presets；停滞降温系数为 0.9

3. **重启策略**
   - 多重启时优先保留**可行解**；均可行时取距离最短者

**与 CVRP 的差异：**
- 可行性判断需模拟整条路线的时间推进（到达 → 等待 → 服务 → 离开）
- 初解多一条 CVRP warm start 路径
- 邻域更少（无 Or-opt 链、cross-exchange），但每步均校验时间窗
